1. 本周问题
前期 Lab1 / Lab2 / Lab3 已经得到的基本判断是：
- Soft Thinking 在长链推理中存在 collapse / OOD 失稳现象；
- 失败往往不是最终答案单点错误，而是中间 reasoning dynamics 逐步漂移、重复、过长、at-limit；
- Lab3 将注意力收缩到 branch point：soft 的有效作用可能不是全程开启，而是集中在少数 reasoning 分叉点。
但 SeLaR 把原本我们的思路做了，所以现在需要研究的是：
- SeLaR 是否已经做掉了 entropy-gated selective soft 这个想法？
- SeLaR 是否已经解决了我们关心的 collapse？
- 如果 SeLaR 没有解决，我们的 contribution 应该从哪里继续收缩？
因此本周做了两组实验：
1. SeLaR reproduction：把 SeLaR 作为 external baseline，检查它在标准数据集上是否稳定提升，以及是否缓解 long-tail/collapse。
2. Lab4 oracle probe：不训练 detector，先做实验，检验在 recoverable branch 局部开启 soft 是否有效。

2.6 判断
SeLaR 与我们的方向有重合，但没有完全覆盖我们的问题。
未解决部分：
- SeLaR 主要依赖 entropy gate，而 entropy 高不等于 branch recoverable；
- SeLaR 没有区分 recoverable branch / degenerative branch / pseudo branch；
- SeLaR 没有直接证明 gate 命中了真正可恢复的推理分叉；
- SeLaR 在错误样本上反而增加长度，说明它可能加剧某些 collapse tail。
因此我们要研究的是：
什么样的 branch 值得 soft，什么样的 branch 不该 soft。

3. 实验二：Oracle
3.1  实验目的
区分：
- recoverable branch：模型处在可恢复分叉处，局部 soft 可能可以帮助重新选择路径；
- degenerative branch：模型已经进入错误/重复/漂移轨道，soft 不能救，甚至会放大 collapse；
- pseudo / late branch：看起来像 branch，但太晚或不是 actionable；
- stable control：理论上不需要 soft 干预的稳定点。
核心假设：
Soft Thinking 的有效作用不是集中在所有高不确定点，而是集中在少数 recoverable branch；对 degenerative / pseudo branch 开 soft 可能无效或有害。
本次实验是人为提供 branch anchor，只检验如果 anchor 已知，局部 soft 是否有因果效果。

4. 阶段性总结
4.1  SeLaR 没有把我们的方向完全做掉
SeLaR 做的是 entropy-triggered reasoning intervention。它说明在不确定处做额外思考是合理方向，但它没有证明：
- entropy 高的位置就是 recoverable branch；
- soft/latent intervention 应该在所有 entropy-high 位置开启；
- 这种 intervention 能稳定避免 collapse；
- 它能减少 long-tail reasoning。
我的复现显示：
- SeLaR accuracy 不稳定；
- SeLaR 会修复一些样本，也会破坏一些样本；
- SeLaR 增加平均长度和错误样本长度；
- 因此 entropy gate 不是足够的 solution。
4.2  Lab4 oracle 实验给出正信号，但还不是最终 solution
Lab4 结果支持：
- full-chain soft 是危险的；
- bounded branch-local soft 更安全；
- recoverable branch 上存在 oracle benefit；
- degenerative branch 不应该 soft。
但 Lab4 也暴露问题：
- pseudo/late bucket 混入了可能 recoverable 的样本；
- stable control 也可能被 short-window perturbation 改善；
- 目前不能证明效果是 branch-specific；
- 不能直接用当前标签训练 recoverability detector。
5.  当前 solution 方向
三阶段 controller：
1. Detect / score recoverability
  - 判断当前局部状态是不是 recoverable branch；
  - 不能只看 entropy；
  - 需要结合局部 dynamics，例如 answer proximity、repetition risk、semantic route switch、top-k diversity、soft embedding drift 等。
2. Selective local soft
  - 只在 recoverable branch 附近短窗口开启 soft；
  - 当前 oracle 结果显示 w16 比 w8 更有效，比 w32 更稳；
  - window 过长会重新引入 collapse。
3. Commit / return hard
  - soft 只用于短暂探索；
  - 一旦跨过 branch 或出现 answer commitment，应返回 hard decoding；
  - 避免 full-chain soft 导致长尾漂移。

1.  下一阶段任务
1.1  实验：shifted/random-window specificity control
下一阶段最重要的是 specificity control。
实验问题：
branch-local soft 的收益是否真的来自 branch anchor，还是任意相近位置开一个短 soft window 都可能有类似收益？
实验设计：
对同一批 anchor，比较：
- hard_only
- always_soft
- branch_w16
- shift_-512_w16
- shift_-128_w16
- shift_+128_w16
- shift_+512_w16
- random_matched_w16
优先子集：
1. positive 中 branch_w16 帮助的样本；
2. negative 中 branch-local soft 伤害的样本；
3. pseudo/late 中 branch-local soft 意外帮助的样本；
4. stable control 中 branch-local soft 意外帮助的样本。
判定标准：
- 如果 true branch window 在 positive 上显著优于 shifted/random，而在 negative/pseudo 上不提升，则说明 recoverability-aware branch anchor 是有效机制；
- 如果 shifted/random 也差不多有效，则说明当前效果更像 local stochastic perturbation；
- 如果 pseudo/late 中 true branch 也显著优于 shifted/random，则需要把这些 pseudo/late 重新标为 recoverable 或 ambiguous-positive。
2. Solution的初步思路
把branch point的样本分类完，然后将到达 branch point 的前面窗口附近的熵、top-k 分布、hidden/soft 表征变化等一系列参数拉出来对比，找可分特征，然后量化一个分类逻辑，实现在线判断 branch point 类型。
